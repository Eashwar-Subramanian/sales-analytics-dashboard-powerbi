# Contributing / Reviewing

I want blunt, practical feedback more than praise.

## Fastest way to help
1) Open **Issues**
2) Create a new issue titled: **Review: <repo-name>**
3) Answer these 5 questions:
- What looks unclear or untrustworthy on first read?
- What would you remove to make this easier to evaluate?
- What is missing for reproducibility?
- What would you expect to see for a junior analytics portfolio?
- Any naming/structure improvements?

## If you submit a PR
- Keep changes small and focused
- Explain the “why” in the PR description
- No secrets, keys, tokens, `.env`, or generated datasets
